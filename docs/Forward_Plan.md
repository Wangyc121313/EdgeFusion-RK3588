**工作分解计划（无时间线，按成熟度推进）**

### 1. 基线治理与评测体系（先做）
目标：先把“评价标准”做稳，避免后续优化无从比较。  
任务：
1. 固化统一测试入口，区分功能测试与压力测试。
2. 固化核心指标计算脚本，确保每次运行都有同一格式输出。
3. 建立回归报告模板，支持横向对比不同版本。
4. 记录硬件环境与参数快照，避免结果不可复现。  
产出物：
1. 稳定的测试脚本集与报告目录规范。
2. 一份长期指标基线文档。  
验收标准：
1. 任意一次测试都能自动产出可比报告。
2. 同场景重复测试的指标波动可解释。

### 2. 模型部署升级主线（重点一）
目标：从“能量化”升级到“量化可控、精度可控、性能可控”。  
任务：
1. 建立量化校准集策略：按光照、距离、遮挡、运动状态分层采样。
2. 做模型敏感度分析：逐层或逐模块定位量化敏感区域。
3. 实施混合精度策略：敏感层保精度，其他层追性能。
4. 引入 QAT 试验分支：重点针对回归严重类别。
5. 做端侧推理 Profiling：算子耗时、内存拷贝、流水阻塞点。
6. 形成部署版本矩阵：精度优先版、均衡版、性能优先版。  
产出物：
1. 模型版本对比表（精度-延迟-功耗）。
2. 量化策略说明文档与复现实验配置。
3. 部署自动化脚本（导出、转换、验证、打包）。  
验收标准：
1. 相比当前基线，在相同精度下时延下降，或在相同时延下精度提升。
2. 长时压测中稳定性不下降。

### 3. 多模态融合升级主线（重点二）
目标：从传统融合升级为“可学习融合”，但保留工程可解释性。  
任务：
1. 保留现有传统融合作为保底路径。
2. 设计轻量学习融合头：输入视觉检测特征、几何特征、时序特征，输出距离修正与融合置信度。
3. 建立融合门控机制：低置信度自动回退传统融合。
4. 加入不确定性估计：为每个目标输出可信区间或风险分数。
5. 优化时序融合：利用轨迹历史做稳定化与异常抑制。
6. 评估不同融合策略在遮挡、交叉、低光场景下的收益。  
产出物：
1. 融合策略对比实验报告（传统、学习、混合门控）。
2. 可解释字段扩展（误差来源、门控原因、回退标记）。
3. 线上融合配置开关与实验配置模板。  
验收标准：
1. 关键场景下距离抖动下降、跟踪连续性提升。
2. 学习融合失效时可平滑回退，不引入灾难性退化。

### 4. 数据闭环与难例挖掘
目标：让系统具备“越跑越好”的能力。  
任务：
1. 构建难例定义规则：漏检、误检、ID 切换、距离突变、时延超阈。
2. 自动抽取难例片段并归档到训练/校验池。
3. 建立伪标签质量分层：高可信、待审查、可丢弃。
4. 构建小规模人工审核闭环，用于关键场景纠偏。  
产出物：
1. 难例样本池与索引。
2. 数据质量报告与清洗策略。  
验收标准：
1. 每次迭代都能明确说明“用了哪些新增数据，带来哪些提升”。
2. 指标提升可归因到具体数据与策略。

### 5. 硬件加速与媒体链路优化
目标：打造可长期稳定运行的边缘实时系统。  
任务：
1. 完整梳理零拷贝路径与内存带宽瓶颈。
2. 建立多线程流水线压测：采集、预处理、推理、编码、发布分段观测。
3. 推流侧加入动态自适应：码率、分辨率、关键帧策略。
4. 完善连接恢复与故障自愈策略。  
产出物：
1. 端到端时延分解看板。
2. 高负载下的稳定性报告。  
验收标准：
1. 压测阶段无明显卡死或不可恢复中断。
2. 热态下指标退化可控且可观测。

### 6. 工程化与可维护性
目标：项目长期可迭代、可协作、可交付。  
任务：
1. 统一配置管理与实验配置版本化。
2. 增加核心模块单测与集成测试。
3. 增加关键路径性能回归测试。
4. 完善架构文档与接口契约。  
产出物：
1. 测试矩阵与自动回归流程。
2. 设计文档与发布说明模板。  
验收标准：
1. 新功能引入后可快速判断是否回归。
2. 新成员可按文档完成环境搭建与基本测试。
